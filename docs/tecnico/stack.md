# Stack utilizada

* [Python](https://www.python.org/) - linguagem de programação escolhida para desenvolver as lógicas do jogo incluindo combate, fisica e colisões, geração de mundo e estruturas de salvamento persistente de dados e de classes robustas e escaláveis. O principal motivo da escolha é pela simplicidade do uso de POO e por ser a linguagem relacionada com o Pydroid3, que nas buscas por uma linguagem para desenvolver pelo android era a que tinha mais recursos para oferecer de todas as linguagens e IDEs testadas.
* [Kivy](https://www.kivy.org/) - famework gráfico cross-plataform usado para as Interfaces do jogo como telas, layouts, imagens, animações. O motivo principal para essa escolha é porque no inicio do projeto o jogo estava sendo produzido apenas em celular, o que pedia de um framework possivel de testar diretamente no android para que o projeto pudesse começar, e esse foi o framework achado para sanar tais necessidades.

RadioRoots nasceu da necessidade de aprender, sem ter os melhores recursos disponíveis para o desenvolvimento, a escolha da stack foi pensada em desenvolvolver apenas com programação(Sem Engine), e com a capacidade de desenvolver pelo android quando outros recursos ainda não estavam disponíveis, as limitações de desempenho de tal stack é conhecida, e no desenvolvimento é pensado como evitar ao máximo essas quedas e entregar boas experiências ao usuario final.
